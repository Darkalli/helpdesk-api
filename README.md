# HelpDesk API — Enterprise Ticket Management System

A robust, secure, and modern RESTful API designed for managing technical support tickets (Help Desk). Built with cutting-edge technologies like **Java 25** and **Spring Boot 4**, this system features a complete role-based authentication flow, ticket lifecycles, and performance reporting.

This repository serves as a professional Back-end portfolio showcase, demonstrating solid concepts of clean code, relational database modeling, and API security.

---

## Key Features

* **Role-Based Access Control (RBAC):** Distinct behavior and permissions for `Admin`, `Agent`, and `Client` using JWT.
* **Ticket Lifecycle Management:** Complete workflow from creation, agent assignment, status updates, to final resolution.
* **Collaboration:** Built-in commenting system inside support tickets.
* **Extensibility:** Categorized ticket classification and automated report generation capabilities.

---

## Tech Stack & Tools

* **Core Language:** Java 25
* **Framework:** Spring Boot 4.0.6

## Domain Model & Entities

The database uses inheritance mapping to manage different user roles starting from an abstract base class:

| Entity | Description |
|---|---|
| `User` (abstract) | Base identity model for `Admin`, `Agent`, and `Client`. |
| `Admin` | System administrator with full management access. |
| `Agent` | Support agent responsible for solving tickets. |
| `Client` | Customer who opens support tickets. |
| `Ticket` | The core support ticket containing logs and history. |
| `Category` | Classification categories to organize tickets. |
| `Comment` | Internal or customer interactions within a ticket. |
| `Report` | Analytical metrics and reports generated by the system. |

## API Endpoints

### Users (`/api/users`)

| Method | Route | Description |
|---|---|---|
| POST | `/api/users` | Register a new user profile |
| GET | `/api/users/{id}` | Fetch a specific user by ID |
| PATCH | `/api/users` | Update user data (Admin only) |
| PATCH | `/api/users/password` | Update current user password |

### Tickets (`/api/tickets`)

| Method | Route | Description |
|---|---|---|
| POST | `/api/tickets` | Create a new support ticket |
| PATCH | `/api/tickets/link` | Assign an agent to a specific ticket |
| PATCH | `/api/tickets/status` | Update ticket status (e.g., IN_PROGRESS) |
| PATCH | `/api/tickets/close` | Permanently close a ticket |
| PATCH | `/api/tickets/comment` | Add a comment interaction to a ticket |
| PATCH | `/api/tickets/change` | Reassign/change the current ticket agent |

### Comments (`/api/comments`)

| Method | Route | Description |
|---|---|---|
| PATCH | `/api/comments` | Edit an existing comment |
| DELETE | `/api/comments/{id}` | Delete a specific comment by ID |
| GET | `/api/comments/{id}/author` | Retrieve the comment author's User ID |

### Categories (`/api/category`)

| Method | Route | Description |
|---|---|---|
| POST | `/api/category` | Create a new ticket category |
| PATCH | `/api/category` | Update category information |
| DELETE | `/api/category/{id}` | Delete a category by ID |
| GET | `/api/category/{id}` | List all tickets belonging to a category |

## How to Run Locally

### Prerequisites
* **Java JDK 25** or higher installed.
* **MySQL Server**

### Database Configuration
The application is pre-configured to automatically create the database if it doesn't exist. Update your credentials in `src/main/resources/application.properties` if needed:

```properties
spring.datasource.url=jdbc:mysql://DBURL/DBName?createDatabaseIfNotExist=true
spring.datasource.username=DBUsername
spring.datasource.password=DBPassword
server.port=8081
```

### Run

```bash
./mvnw spring-boot:run
```

### Swagger UI

Go to `http://localhost:8081/swagger-ui/index.html#/` after start.

ctrl + c to finish
